* Para iniciar variáveis de ambiente, usar arquivo /etc/bash.bashrc

* Para iniciar outras coisas usar o arquivo /etc/rc.local
    * Inicialmente este arquivo não existe no ubuntu
        * Para criar o arquivo execute:
        $ printf '%s\n' '#!/bin/bash' 'exit 0' | sudo tee -a /etc/rc.local
        $ sudo chmod +x /etc/rc.local
    * Já vem no ubuntu, um serviço chamado "rc-local"
        * Mas quando vai habilitar o serviço, não funciona
        * Segui estes tutoriais:
            * https://www.vivaolinux.com.br/dica/rclocal-na-systemd-script-de-execucao-na-inicializacao-do-sistema
            * https://www.linuxbabe.com/linux-server/how-to-enable-etcrc-local-with-systemd
        * Criei o arquivo "/etc/systemd/system/rc-local.service"
        * Daí é só habilitar e inicar o serviço:
            $ sudo systemctl enable rc-local
            $ sudo systemctl start rc-local.service
            $ sudo systemctl status rc-local.service



* Conteúdo do "/etc/systemd/system/rc-local.service"
[Unit]
Description=/etc/rc.local Compatibility
ConditionPathExists=/etc/rc.local

[Service]
Type=forking
ExecStart=/etc/rc.local start
TimeoutSec=0
StandardOutput=tty
RemainAfterExit=yes
SysVStartPriority=99

[Install]
WantedBy=multi-user.target
